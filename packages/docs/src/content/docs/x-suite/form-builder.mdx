Visual form designer (FormBuilder) is planned for v2.

## Quick Start

```tsx
import { FormRenderer } from '@mfe/x-form-builder';

<FormRenderer schema={schema} onSubmit={handleSubmit} />
```

## Core Components

| Component | Purpose |
|-----------|---------|
| `FormRenderer` | Renders forms from a `FormSchema` |
| `FormPreview` | Live form preview pane |
| `FieldRenderer` | Single field renderer (with 12 built-in field types) |
| `FormSectionComponent` | Grouped field section |
| `MultiStepForm` | Multi-step wizard form |
| `FormSummary` | Review summary before submission |
| `RepeatableFieldGroup` | Dynamic array of field groups |

## Field Registry

Register custom field renderers:

```tsx
import { FieldRegistryProvider, createFieldRegistry } from '@mfe/x-form-builder';

const registry = createFieldRegistry();
registry.register('color-picker', MyColorPicker);

<FieldRegistryProvider registry={registry}>
  <FormRenderer schema={schema} onSubmit={handleSubmit} />
</FieldRegistryProvider>
```

## Built-in Field Renderers

`TextFieldRenderer`, `NumberFieldRenderer`, `SelectFieldRenderer`,
`MultiselectFieldRenderer`, `CheckboxFieldRenderer`, `RadioFieldRenderer`,
`SwitchFieldRenderer`, `TextareaFieldRenderer`, `DateFieldRenderer`,
`TimeFieldRenderer`, `FileFieldRenderer`, `CustomFieldRenderer`

## Schema Format

Forms are defined as JSON schemas:

```json
{
  "title": "Contact Form",
  "fields": [
    { "type": "text", "name": "fullName", "label": "Full Name", "required": true },
    { "type": "email", "name": "email", "label": "Email" },
    { "type": "select", "name": "department", "label": "Department",
      "options": ["Engineering", "Sales", "Support"] },
    { "type": "textarea", "name": "message", "label": "Message" }
  ]
}
```

## Conditional Logic

Show or hide fields based on other field values:

```json
{
  "type": "text",
  "name": "otherReason",
  "label": "Please specify",
  "condition": { "field": "reason", "eq": "other" }
}
```

## Hooks

| Hook | Purpose |
|------|---------|
| `useFormSchema` | Form state from a `FormSchema` |
| `useZodForm` | Zod-powered form validation |
| `useConditionalLogic` | Evaluate conditional visibility rules |
| `useAsyncValidation` | Async field validation |
| `useMultiStepForm` | Multi-step wizard navigation and state |

## Validation

Built-in validators include `required`, `min`, `max`, `pattern`,
`email`, `url`, and custom async validators.

Zod integration is available via `createZodValidator`, `toZodSchema`,
and `fromZodSchema` utilities.
