# Support Model

## Issue Triage
| Priority | Response | Resolution |
|----------|----------|------------|
| P0 Critical | 4 hours | 24 hours |
| P1 High | 1 business day | 1 week |
| P2 Medium | 3 business days | 2 weeks |
| P3 Low | 1 week | Best effort |

## Channels
- **Bug Reports**: GitHub Issues with reproduction
- **Feature Requests**: GitHub Discussions
- **Security**: security@company.com (see SECURITY.md)

## Release Cadence
- **Patch**: Weekly (bug fixes, security)
- **Minor**: Bi-weekly (features, non-breaking)
- **Major**: Quarterly (breaking changes, deprecation removal)

## SLA
- Design system unit tests: >95% pass rate
- Critical accessibility: WCAG 2.1 AA compliance
- Performance: <16ms render for core primitives
