## Tech Stack Decision

| Concern | Choice | Rationale |
|---------|--------|-----------|
| Framework | Nextra 4 (Next.js 14 + MDX) | First-class MDX, file-system routing, built-in search |
| API pages | Auto-generated from `.doc.ts` entries | Single source of truth; no manual sync |
| Live playground | Sandpack (CodeSandbox) | In-browser bundler, supports TS + React |
| Search | Pagefind (static) initially, Algolia DocSearch at scale | Zero-config start, upgrade path when traffic warrants |
| Hosting | Vercel (preview per PR) | Automatic preview deployments for doc PRs |
| Versioning | docs/v{major} branches | One live version initially; branch when breaking changes ship |

## Page Templates

### Component Page Layout

Every `/components/{name}` page follows a consistent layout:

1. **Header** -- component name, package badge, quality score badge
2. **Import statement** -- copy-to-clipboard code block
3. **Overview** -- one-paragraph description + hero example
4. **Props table** -- auto-generated from `.doc.ts` apiItem.props
5. **Variants** -- visual grid of variant axes (size, color, etc.)
6. **Examples** -- categorized code examples with live preview
7. **Playground** -- Sandpack editor with editable props
8. **State model** -- state diagram (if component has stateModel)
9. **Accessibility** -- ARIA roles, keyboard navigation notes
10. **Related** -- links to related components and patterns

### Pattern Page Layout

Every `/patterns/{name}` page follows:

1. **Problem statement** -- what the user is trying to build
2. **When to use** -- decision criteria
3. **Architecture** -- composition diagram showing which components combine
4. **Step-by-step** -- numbered build guide with code at each step
5. **Complete example** -- full working code
6. **Variations** -- alternative approaches for different requirements

## Content Authoring Workflow

1. Author writes MDX in `docs/content/{section}/{page}.mdx`
2. Component API pages are generated by `scripts/generate-api-docs.ts`
3. PR triggers Vercel preview build
4. Reviewer checks rendered preview
5. Merge to main deploys to production docs site

## Metrics

Track via Vercel Analytics + Pagefind query logs:

- **Search miss rate** -- queries returning zero results (signals missing docs)
- **Page bounce rate** -- pages users leave immediately (signals poor content)
- **Top search queries** -- prioritize docs for most-searched topics
- **Time to first interaction** -- playground engagement
