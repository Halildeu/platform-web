# Support Model

## Issue Reporting

Report issues via [GitHub Issues](https://github.com/Halildeu/platform-ssot/issues).

### Issue Priority Levels

| Priority | Definition | Response Target | Resolution Target |
|---|---|---|---|
| P0 — Critical | Production down, data loss | 4 hours | 24 hours |
| P1 — High | Major feature broken, no workaround | 1 business day | 3 business days |
| P2 — Medium | Feature degraded, workaround exists | 3 business days | Next sprint |
| P3 — Low | Minor issue, cosmetic | 1 week | Backlog |

### Issue Template

When reporting an issue, include:
1. Component name and version
2. Environment (React version, browser, OS)
3. Minimal reproduction steps
4. Expected vs actual behavior
5. Screenshots if visual

## Release Channels

| Channel | Cadence | Stability |
|---|---|---|
| `latest` | Every 2-4 weeks | Production-ready |
| `beta` | Weekly | Feature-complete, minor bugs possible |
| `canary` | Daily | Unstable, for early testing |

## Known Issues

Track known issues and workarounds at [Known Issues](https://github.com/Halildeu/platform-ssot/labels/known-issue).
